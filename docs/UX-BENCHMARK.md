## 2. 우리 사이트 클릭 동선 매핑

### 현재 주 진입 → 도착 흐름

| 시작 | 클릭 | 결과 | 평가 |
|------|------|------|------|
| 랜딩 hero | "🗺️ 지도로 시작하기" | 대시보드 (지도 + 우측 패널) | ✅ 명확 |
| 랜딩 hero | "📊 지표별로 보기" | 풀스크린 탐색 페이지 | ✅ 명확 |
| 랜딩 Bento | "시군 랭킹" | 풀스크린 랭킹 페이지 | ✅ 명확 |
| 지도 | 시군 클릭 | 우측 패널: 점수·레이더·지표·자율지표 | ✅ 명확 |
| 지도 줌 11+ | 읍면 클릭 | 우측 패널 in-place 교체 | 🟡 패턴은 OK이나 어떤 정보를 볼 수 있는지 미리 안내 부족 |
| 시군 패널 | 지표 카드 클릭 | 탐색 페이지 + 그 시군 pin | ✅ 명확 |
| 탐색 페이지 | 좌 카테고리 클릭 | 우측 상세 갱신 | ✅ 명확 |
| 탐색 페이지 | "🗺️ 지도에서 보기" | 지도 화면 + 해당 지표 코로플레스 | ✅ 명확 |

### 약점

1. **읍면 단위 비교 진입** — 시군 안에서 읍면 비교 UI 없음 (피드백 #2 — Commit 6 에서 추가)
2. **시사점 텍스트 부족** — 점수·차트만 보이고 "그래서?" 가 약함 (피드백 #3 — Commit 3 에서 추가)
3. **사용자 유형 분리 부재** — 일반·관리자 모두 같은 화면 (피드백 #6 — Commit 4 에서 추가)
4. **랜딩에 반복되는 작은 라벨** — "WHY", "REGIONS" 같은 uppercase kicker × 4 = 편집 스캐폴딩 인상 (impeccable detect 반영)

## 3. 차용한 패턴 (이미 적용됨)

| 패턴 | 출처 | 우리 적용처 |
|------|------|----------|
| 3-column thematic map | SGIS 통계주제도 | 지표 탐색 페이지 (`#explore-screen`) |
| 좌측 카테고리 트리 (collapsible) | SGIS / KOSIS | 탐색 페이지 사이드바 |
| 시군 클릭 → 정보 카드 | 모든 정부 포털 | 지도 우측 패널 |
| 5분위 코로플레스 | 통계청 표준 | 지도 색상 분류 |
| breadcrumb 드릴다운 | UPIS · 행정자치부 | 시군 › 읍면 › 행정리 |
| 비교 모드 (최대 5개) | KOSIS 통계표 비교 | 시군 비교 탭 |

## 4. 다음 Commit 별 클릭 동선 변화

### Commit 3 — 시사점 카드 추가
- **새 진입**: 지표 카드 (예: L1)에 시사점 한 줄 미리보기 + 클릭 → 상세 정책 제안
- **결과**: 사용자가 점수 → "그래서 어떻게" 까지 같은 화면에서 확인

### Commit 4 — 일반/관리자 토글
- **새 진입**: 헤더에 토글 (👁️ 일반 / ✏️ 관리자)
- **관리자**: 시군 패널 안에 manual 입력 폼 + 자율지표 선정 체크박스
- **결과**: 데이터 책임자 / 일반 사용자 화면 분리

### Commit 6 — 남양주 읍면 비교
- **새 진입**: 남양주 시군 패널 안에 "📍 24개 읍면 비교" 섹션
- **클릭**: 읍면 다중 선택 → 막대 차트 비교 + 지도 강조 동기화
- **결과**: 시군 안에서 더 깊은 비교가 가능

### Commit 7 — AI 해석 카드
- **새 진입**: 시군 패널과 탐색 페이지에 "💡 AI 해석" 카드
- **결과**: 정량 (점수) + 정성 (해석) 균형. 일반 사용자도 시군 강점·약점 이해

## 5. 클릭 동선 향후 검토 항목

- [ ] 첫 방문자가 "지도로 시작하기"와 "지표별로 보기" 중 어느 쪽을 더 자주 누르는지 (A/B 또는 사용자 인터뷰)
- [ ] 읍면 패널 진입 후 "어디로 더 깊이 갈지" 명확한 다음 행동 안내
- [ ] 관리자 모드의 학습 곡선 (첫 진입 시 가이드 툴팁)
- [ ] 시사점·AI 해석을 사용자가 신뢰하는지 (출처·근거 표시)
