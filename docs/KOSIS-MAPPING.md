# KOSIS API ↔ 21개 농촌다움 지표 매핑 마스터

> 이 문서는 농촌다움 지표 21개(공통 11 + 남양주 자율 10) 각각이 어떤 KOSIS 통계표와 산식으로 구성되는지를 정리한 마스터.  
> `fetch_kosis.py`에 새 테이블을 추가하기 전, 또는 manual 입력을 채우기 전 이 문서를 참조.

**최종 갱신**: 2026.05  
**테스트 환경**: KOSIS Open API v3 (`statisticsParameterData.do`)

---

## 1. Tier 분류 요약

| Tier | 정의 | 항목 수 |
|------|------|------|
| **1** | KOSIS API로 즉시 수집 가능 — `fetch_kosis.py`에 활성화됨 | 2 (인구·세대수) |
| **2** | KOSIS API에 데이터는 있으나 차원 코드·40k 셀 제약으로 수집 보류 | 6 |
| **3** | KOSIS 외 출처 — region-meta.json의 `manual` 층에 수동 입력 | 13 |

## 3. 활성화된 KOSIS 테이블 (Tier 1)

### 3-1. DT_1B040A3 — 시군구별 주민등록인구

| 항목 | 값 |
|------|---|
| orgId | 101 |
| 주기 | 월(M), 최신 13개월 (현재 + 전년동월) |
| 차원 | C1 = 행정구역코드 (5자리) |
| ITM_IDs | T20=총인구수, T21=남자, T22=여자 |
| 파라미터 | `objL1=ALL&itmId=ALL&prdSe=M&newEstPrdCnt=13` |
| 산출 | `raw.population`, `raw.population_prev` → `computed.L1_pop_growth_rate` |

### 3-2. DT_1B040B3 — 시군구별 주민등록세대수

| 항목 | 값 |
|------|---|
| orgId | 101 |
| 주기 | 월(M), 최신 1개월 |
| 차원 | C1 = 행정구역코드 (5자리) |

## 4. 비활성 KOSIS 테이블 (Tier 2 — 시도 시 발생한 오류)

### 4-1. DT_1B26001 — 인구이동 (L3 입력 필요)
- 응답 차원: C1=시도, C2=성별, C3=연령. **시군구 breakdown 없음**.
- `objL1='41'` → C1="41" 단일값(경기도 전체)만 반환. 시군구 데이터 미존재.
- **대안**: KOSIS 웹 → 시군구별 인구이동 (다른 표 ID 필요) → Excel 다운로드 → manual 입력

### 4-2. DT_1C81 — GRDP (W4 입력 필요)
- 응답 차원: C1, C2 — **모두 시도 단위만**.
- `objL1=ALL` → 17개 시도, 시군구 없음.
- **대안**: KOSIS 「지역소득」 → 시군구별 GRDP 표는 별도 ID. KOSIS 웹 검색 필요.

### 4-3. DT_1B81A21 — 합계출산율
- 응답 차원: C1 = KOSIS 내부 시도코드(11=서울, 31=경기). 행안부 코드(41) 아님.
- 시군구 breakdown 없음.

### 4-4. DT_1IN1502 — 연령별 인구 (L2 노령화지수)
- ERR 31 (40,000셀 초과). 모든 시군구 × 연령 × 항목 = 큰 데이터.
- **해결책**: 특정 ITM_ID(65세이상, 0-14세)만 지정 → KOSIS 웹에서 ITM_ID 코드 확인 필요.

### 4-5. DT_1K51003 — 전국사업체조사 (W2, W8 입력)
- ERR 31. 산업분류 × 시군구 × 항목 = 큰 데이터.
- **해결책**: 산업분류 전산업 합계 코드 지정 → 코드체계 확인 필요.

### 4-6. DT_1B040M1 — 읍면동별 인구
- ERR 31 (3차원 모두 ALL 시).
- **해결책**: 경기도 한정 객체 코드 형식 확인 (현재 `41` `4100000000` 모두 ERR 21).

## 6. 산식 자동 계산 — `compute_indicators()` 매핑

| computed key | 산식 | 필요 raw 필드 | 단위 |
|-------------|------|--------------|------|
| `L1_pop_growth_rate` | (현재-전년)/전년×100 | `population`, `population_prev` | % |
| `L3_net_migration_rate` | (전입-전출)/인구×1000 | `inflow`, `outflow`, `population` | ‰ |

새 산식 추가 시 `scripts/fetch_kosis.py`의 `compute_indicators()` 함수에 추가.  
필요한 raw 필드가 모두 존재할 때만 계산되며, 누락 시 해당 computed 키는 생성되지 않음.

## 7. region-meta.json 3-layer 스키마

```jsonc
{
  "sigun": {
    "namyangju": {
      "raw":      { /* KOSIS 원본 — fetch_kosis.py가 매번 갱신 */ },
      "computed": { /* 산식 계산값 — raw로부터 결정론적 */ },
      "manual":   { /* 수동 입력값 — 스크립트가 보존 */ }
    }
  }
}
```

- `raw`/`computed`: `fetch_kosis.py` 실행 시마다 덮어씀
- `manual`: `load_existing_manual()`이 기존 파일에서 읽어 보존
- 사용자 직접 입력 예시:
  ```json
  "manual": {
    "W3_fiscal_independence": {
      "value": 38.6,
      "year": "2024",
      "source": "지방재정365",
      "updated_by": "user",
      "updated_at": "2026-05-17"
    }
  }
  ```
