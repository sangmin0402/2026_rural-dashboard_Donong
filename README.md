## 1. 프로젝트 개요

**법적 배경**  
농촌공간 재구조화 및 재생지원에 관한 법률(2023) 시행에 따라 경기도 내 농촌 시군은 각자 기본계획을 수립해야 함. 이 플랫폼은 계획 수립을 지원하기 위한 데이터 시각화 도구임.

**대상 시군** (15개)  
남양주시, 가평군, 양평군, 여주시, 이천시, 안성시, 양주시, 포천시, 동두천시, 광주시, 화성시, 평택시, 오산시, 용인시, 하남시

**수업**: 지역활성화시스템론 (2026)

## 2. 주요 기능

### 🌐 랜딩 포털 (첫 화면)
- 플랫폼 소개 및 농촌다움 개념 설명
- 경기도 SVG 미니맵 + KPI 카운터 (15개 시군 · 21개 지표)
- 5대 권역 소개 (보고서 표 2-36 기준)
- 6개 진입 카드: 지도 / 비교 / 시나리오 / 랭킹 / **📊 지표 탐색** / 남양주
- 헤더 로고 클릭 또는 브라우저 뒤로가기로 첫 화면 복귀

### 📊 지표 탐색 페이지 (NEW)
- 풀스크린 오버레이 — 랜딩 카드 또는 `#explore` 해시로 진입
- **좌측**: 21개 지표 목록 (공통 11 + 자율 10), 검색·카테고리 탭으로 필터
- **우측 상세** (지표 선택 시):
  - 산식·출처 (KOSIS/SGIS computed 키와 매핑)
  - 15시군 비교 표 — 시군명·값·raw 데이터·5분위 등급, 정렬 가능
  - 시군별 막대 차트 (Chart.js)
  - SVG 미니 코로플레스 (클릭 시 해당 시군으로 이동)
  - "🗺️ 지도에서 이 지표로 보기" — 지표 선택 상태로 지도로 이동
- 지도 우측 패널의 지표 카드(공통·자율 모두) 클릭 시에도 자동 진입

### 🗺️ 지표 지도
- Leaflet.js 기반 코로플레스 지도
- 지표 선택으로 실시간 색상 갱신 / 5분위(quantile) 색상 분류
- **줌 연동 레이어 자동 전환**: 기본 → 시군, 줌 11+ → 읍면, 줌 13+ → 행정리
- **시군명 라벨** 항상 표시 / 줌인 시 읍면·행정리 이름도 지도 위에 상시 표시
- **지도 컨텍스트 HUD**: 현재 화면 중심부가 어느 시군/읍면인지 좌하단에 실시간 표시
- **선택 토스트**: 시군·읍면·행정리 클릭 시 상단 중앙에 레벨 구분 토스트 알림
- **시군 경계선 오버레이**: 줌인해도 시군 경계가 사라지지 않음
- **지도/패널 드래그 리사이저**: 좌우 비율을 자유롭게 조절 가능
- **베이스맵 선택 (7종)**: Positron · Voyager · Dark Matter · OSM · Esri Topo / Gray / 위성

### 📊 레이더 차트
- 선택 시군의 삶터·일터·쉼터 종합점수 시각화
- 삶터/일터/쉼터/시나리오 탭 전환

### 📈 KOSIS 실제 통계
- 시군 선택 시 레이더 차트 아래에 **실제 KOSIS 통계** 카드 표시
- 현재 수집 항목: 총인구, 세대 수 (2026년 4월 기준, 주민등록인구통계)
- 데이터가 없는 항목은 카드 자체가 표시되지 않음 (graceful hide)

### ⚖️ 시군 비교
- 최대 5개 시군 동시 선택, 지표별 막대 차트 비교

### 🔮 시나리오 분석
- 정책 레버 5종 슬라이더 조정
- before/after 레이더 차트 비교 (L4, W6, W7, R4, R6 적용)

### 🥇 시군 랭킹
- 종합·삶터·일터·쉼터 4개 탭으로 15개 시군 순위 비교
- 1·2·3위 포디움 + 4~15위 리스트 (점수 진행 바)

### 📚 지표 가이드
- 공통지표 11개 + 시군 자율지표(현재 남양주 8개)
- 지표 정의·단위·기준연도·공간단위·산식 + 방향 시각 표기

### 🔍 기능별 분석
- **통과형 식별** → 산점도 (인구순이동률 × 재정자립도, 4 사분면)
- **체류 전환** → 히트맵 (시군 × 정주지표 6개 매트릭스)

### 📍 지역 드릴다운 (시군 → 읍면 → 행정리)
- 줌 11+: 읍면 클릭 → 우측 패널 in-place 교체
- 줌 13+: 행정리 클릭 → 행정리 상세 패널
- 상단 breadcrumb (`🏙️ 남양주시 › 📍 진건읍 › 🏘️ 신월리`) — 단계 클릭으로 자유 이동
- 읍면 패널: 인구·면적·가구·토지이용·성격 (KOSIS 연동 또는 예시 데이터 배지)
- 행정리 패널: 소속 읍면·법정리 코드·대표 좌표·개략 면적

### 🤖 AI·해석 기능 (2026-05-31 신규)
- **실시간 LLM 해석 카드**: 시군·읍면의 "🤖 실시간 AI 해석" 버튼 → 지표·비전 점수·발화 트리거 근거로 LLM이 맞춤 해석 생성 (MindLogic UOS Gateway · Cloudflare Worker 프록시)
- **"AI에게 물어보세요" 챗**: 우하단 상시 버튼 → 지금 보는 화면·지표에 자유 질문. **순위·등급·평균은 JS가 계산해 전달, LLM은 서술만 → 환각 차단**. 모델 `gpt-5.4-nano`(저비용)
- **정적 지표 해석 블록** (즉시·무료): 지표 탐색에 "해석 → 평가(등급·순위) → 정책" 3줄 자동 표출
- **조회자 / 읍면담당자 토글** (#6): 조회자=이해 중심, 읍면담당자=담당자 액션·**읍면 현장조사값 직접 입력 폼**·편집까지
- **2035 비전 달성률 카드**: 「2035 남양주 도시기본계획」 목표 대비 달성률 (공원면적 24.5% ~ 고용 87%) + 삶/일/쉼 영역 점수
- ⚠️ **API 키는 클라이언트에 없음** — Worker secret에만 보관 (정적 사이트 보안 원칙 준수)

### 📋 데이터 스키마 (3-layer)

`region-meta.json`은 세 층으로 분리되어 갱신·수동 입력을 동시 지원:

| 층 | 의미 | 사용자가 편집? |
|----|------|----------------|
| **raw** | KOSIS API 원본 | ❌ (`fetch_kosis.py`가 갱신) |
| **computed** | 산식 자동 계산값 (예: L1 인구증가율) | ❌ (raw로부터 자동 재계산) |
| **manual** | 사용자 직접 입력 (예: 재정자립도, 국가유산) | ✅ (스크립트 재실행 시 보존됨) |

### 4-1. 0427 확정 데이터·현장조사 (2026-05)

- **기준 정의**: 상위 폴더의 `공통지표_확정안+남양주 자율지표_0427.xlsx` → `dat/indicator-reference.json` (지표 메타·엑셀 ID ↔ 앱 키 alias 포함).
- **현장조사**: 동일 엑셀 `7_현장조사` 시트 → `dat/field-survey-meta.json` (KOSIS/SGIS 통계와 분리). 남양주시 패널에 **「현장조사 입력/관리」** 블록으로 표시.
- **로컬 수치 반영**: `0427_데이터` 폴더를 읽어 `scripts/import_0427_data.py`가 `region-meta.json`에 병합 (`local0427:*`). `fetch_kosis.py` / `fetch_sgis.py` 실행 뒤 **같은 스크립트를 한 번 더** 돌리면 API 수집값을 유지한 채 로컬 확정값을 다시 덮어씀.
- **갭 리포트**: `dat/data-gap-report.json` — API/로컬 파싱 한계·현장조사 필요 여부 요약 (저장소에 포함; **API 키는 파일에 넣지 않음**).
- **엑셀 ↔ 웹 키 주의**: 엑셀 자율 번호와 앱 키가 한 칸 어긋난 항목이 있음(예: 엑셀 R4 수질 → 앱 **R5**, 엑셀 R5 공원·면적 → 앱 **R6**, 엑셀 W9 체험 파일명 → 앱 **R4** 등). 상세는 `indicator-reference.json`의 `aliases`/`source_key` 참고.

### 4-4. 실시간 LLM AI 해석 (#5, `feat/0531-field-survey-vision`)

정적 텍스트였던 AI 해석을 **MindLogic UOS Gateway**(OpenAI 호환) 실시간 LLM으로 확장. 시군·읍면 카드의 **"🤖 실시간 AI 해석" 버튼** 클릭 시, 지표·비전 적합도·발화 트리거를 근거로 LLM이 맞춤 해석(헤드라인·강점·약점·정책권고·우선조치)을 생성한다.

- **보안**: 정적 GitHub Pages엔 비밀키를 둘 수 없어 **Cloudflare Worker 프록시**(`worker/`)가 키를 보관(`wrangler secret`)하고 게이트웨이를 대신 호출. 클라이언트는 키 없이 구조화 컨텍스트만 전송. 배포·로컬 실행은 [`worker/README.md`](worker/README.md).
- **설정**: `js/config.js`의 `LLM_PROXY_URL`에 배포된 Worker 주소 기입. **비우면 자동으로 사전 작성 텍스트로 폴백**(무중단).
- **캐시·비용**: sessionStorage 캐시(같은 지역 즉시) + "↻ 재생성" 버튼. 모델 `gpt-5.4-nano`(저비용, Worker 상수 1줄로 교체). ⚠️ 키는 어떤 커밋 파일에도 없음(`config.js`엔 공개 URL만).

### 4-5. "AI에게 물어보세요" 대화형 Q&A (#5-2)

화면 우하단 **상시 "🤖 AI에게 물어보세요" 버튼** → 채팅. 지금 보는 화면·지표에 대해 자유 질문("이거 뭐야?", "다른 지역 대비 어때?", "시군 내 몇 등?")하면 답한다.
- **"팩트는 JS가 계산, LLM은 말로만"**: 순위·등급·평균을 클라이언트(`sigunIndicatorFact`/`eupIndicatorFact`)가 계산해 facts로 넘기고 LLM은 인용만 → **순위 환각 차단**. facts에 없으면 "데이터에 없다"고 답함.
- 모델 `gpt-5.4-nano`(문답당 ~350토큰), facts-only 프롬프트. Worker `POST /ask`.
- **정적 해석 블록**(즉시·무료): 지표 탐색 상세에 "해석→평가(등급·순위)→정책" 3줄(`indicator-insights.json` 재사용). LLM 비용 0.

### 4-6. 조회자/읍면담당자 토글 · 읍면 입력폼 · 2035 달성률 · 정책연결 (#6 외)

- **조회자/읍면담당자 토글**(#6): 기존 일반/관리자 토글을 역할 기반으로 재정의(`initViewModeToggle`·`.admin-only`). 조회자=해석·평가·비전 이해 중심, 읍면담당자=거기에 "다음 단계(담당자 액션)·추가 필요 데이터·편집"까지.
- **읍면 현장조사 직접 입력 폼**: 읍면담당자 모드에서 읍면 선택 시 8개 현장지표(L4·L6·W5·W6·W7·W9·R6·R7) 직접 입력(`renderEupFieldEditPanel`). 입력값은 `state.eupOverrides`+localStorage 저장, `getEupIndicator`가 **최우선**(출처 `담당자입력`)으로 읽어 **비전·트리거·시사점 즉시 재계산**.
- **2035 비전 달성률**(박세희 자료): `dat/namyangju-vision-2035.json` + `renderVision2035Card`. 「2035 도시기본계획」 절대 목표 대비 달성률(읍면 상대 visionScore와 보완). 7수치(공원 24.5%↓~고용 87%) + 4방향성 + 삶/일/쉼 영역점수(쉼터 34 최저).
- **정책 라이브러리 → 챗 연결**: 챗이 정책 질문 시 준비된 정책(지표 등급정책 + 트리거 정책 + 2035 달성률)을 인용(`attachPolicyHints`·`attachVision2035Facts`).
- **AI 말투**: 부드러운 존댓말(`TONE_RULE`).

### ⚠️ 데이터 영속성 — 백엔드 필요 (최우선)
현재 사이트는 **정적 GitHub Pages**라 서버가 없어, 사용자가 입력·편집한 값이 **그 브라우저의 localStorage에만** 저장된다. 즉 **다른 사람·다른 기기에서는 보이지 않고, 브라우저 데이터를 지우면 사라진다.** 해당 항목:

- 자율지표 선정, 화면 모드(조회자/읍면담당자)

→ **여러 담당자가 공유하거나 영구 저장하려면 별도 백엔드가 필요하다.** 권장: **Supabase**(PostgreSQL + RLS) 또는 Cloudflare D1 — 읍면×지표×연도 테이블에 `source/n/method/updated_by` 메타와 함께 저장하고, 읍면담당자 인증(읍면별 권한)을 붙이면 실제 운영 가능. (LLM 프록시용 Cloudflare Worker가 이미 있으므로 동일 인프라에 D1을 붙이는 경로가 가장 단순.)
